# RepoSage

[English](./README.md) | [š«ÇńŻôńŞşŠľç](./README.zh.md)

**Code intelligence for ObjC / Swift / Android Java repos ÔÇö dual-layer docs + MCP server.**

RepoSage indexes your mobile codebase into a knowledge graph, then exposes it through:

- **Human-readable Markdown wiki** ÔÇö architecture overview, per-module docs, Mermaid diagrams (generated by Claude)
- **Agent-optimized JSON index** ÔÇö compressed symbol table, call graph, module topology (no LLM needed, minimal tokens)
- **MCP server** ÔÇö 7 tools that let Claude Code, Cursor, and other AI agents query the knowledge graph directly

> Built by combining the best of [GitNexus](https://github.com/abhigyanpatwari/GitNexus) (knowledge graph + MCP) and [DeepWiki](https://github.com/AsyncFuncAI/deepwiki-open) (LLM-generated docs + RAG Q&A), with first-class support for Objective-C, Swift, and Android Java.

---

## ÔťĘ Features

| Feature | Description |
|---------|-------------|
| **ObjC / Swift / Java parsing** | tree-sitter AST ÔÇö extracts classes, methods, protocols, properties, call chains, inheritance |
| **Knowledge graph** | SQLite graph DB ÔÇö symbols, CALLS / EXTENDS / IMPLEMENTS / CONFORMS_TO relations |
| **Dual-layer output** | Human wiki (Markdown) + Agent index (JSON/YAML) in the same repo |
| **MCP server** | 7 tools: `search`, `symbol_context`, `find_callers`, `impact`, `module_overview`, `execution_flow`, `ask` |
| **Semantic search** | ChromaDB + all-MiniLM-L6-v2 embeddings, hybrid with full-text BM25 |
| **Real-time updates** | watchdog file watcher, debounced 300ms incremental re-index |
| **Claude-powered wiki** | Per-module docs + ARCHITECTURE.md generated by Claude API |
| **RAG Q&A** | Ask natural-language questions about your codebase |

### Manual Install

### Prerequisites

- Python Ôëą 3.10 (3.11 recommended)
- `ANTHROPIC_API_KEY` ÔÇö required for wiki generation and the `ask` tool

### Install

```bash
# 1. Clone this repo
git clone https://github.com/your-org/reposage.git
cd reposage

# 2. Create a virtual environment (Python 3.11 via uv ÔÇö recommended)
curl -LsSf https://astral.sh/uv/install.sh | sh
~/.local/bin/uv venv --python 3.11
source .venv/bin/activate   # Windows: .venv\Scripts\activate

# 3. Install dependencies
uv pip install tree-sitter==0.21.3 tree-sitter-languages tree-sitter-swift \
    mcp anthropic chromadb watchdog click rich pyyaml fastapi uvicorn

# 4. Install reposage itself
pip install -e .
```

### Index your repo

```bash
# Full index (parse + embed + generate wiki)
reposage analyze /path/to/your/repo

# Skip wiki generation (faster, no API key needed)
reposage analyze /path/to/your/repo --skip-wiki

# Skip embedding (fastest, search will be FTS-only)
reposage analyze /path/to/your/repo --skip-wiki --skip-embed
```

### Connect to Claude Code

```bash
claude mcp add reposage -- /path/to/.venv/bin/python -m reposage mcp --repo /path/to/your/repo
```

Restart Claude Code. The following tools are now available in your session:

```
search("voice search button")

impact("UserService", direction="upstream")
symbol_context("PFBSearchHomeVC")
module_overview("Authentication")
execution_flow("loginButtonTapped")
ask("How does the keyboard animation work?")
```

## ­čĄľ MCP Tools Reference

Connect with `claude mcp add reposage -- python -m reposage mcp --repo <path>` and use these tools:

### `search`
Hybrid search (full-text + semantic) across all symbols.

```
search(query="keyboard animation", limit=10, language="objc")
```

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `query` | string | required | Natural language or keyword query |
| `limit` | int | 15 | Max results |
| `language` | string | ÔÇö | Filter: `objc`, `swift`, `java` |

---

### `symbol_context`
360┬░ view of a symbol: callers, callees, parent class, module, doc comment.

```
symbol_context(name="keyboardWillChangeFrame:", file="SearchHome/PFBSearchHomeVC.m")
symbol_context(id="5b01b21959040503")   # exact lookup via ID from search results
```

---

### `find_callers`
Find all code locations that call a given method or function.

```
find_callers(name="viewDidLoad", depth=2)
```

---

### `impact`
Blast-radius analysis before editing a symbol.

```
impact(target="PFBSearchHomeVC", direction="upstream", depth=3)
```

| `direction` | Meaning |
|-------------|---------|
| `upstream` | Who depends on this? (what will break) |
| `downstream` | What does this call? |
| `both` | Both directions |

---

### `module_overview`
Files, exported symbols, and description for a module (top-level directory).

```
module_overview(name="PFBWXSearch")
```

---

### `execution_flow`
Trace execution from an entry point, following CALLS edges BFS.

```
execution_flow(entry_point="loginButtonTapped", depth=5)
```

---

### `ask`
RAG-powered Q&A. Searches the symbol index + wiki docs, then answers with Claude.

```
ask(question="How does the voice search feature work end to end?")
ask(question="Which classes handle keyboard frame changes?")
```

> Requires `ANTHROPIC_API_KEY`.

### Agent-layer design

The `.reposage/` JSON files are designed for **minimal token consumption**:

- Each symbol entry: `~80 bytes` (id, name, type, file, line, signature truncated to 120 chars)
- Relations: source_id Ôćĺ target_name + type + confidence
- Modules: name, file list, exported symbols list
- A 750-file ObjC repo produces `~1.2MB` total index (compresses to ~200KB)

## ­čöž Models & Configuration

### LLM for wiki generation and `ask` tool

By default RepoSage uses **Claude** via the Anthropic API.

```bash
export ANTHROPIC_API_KEY=sk-ant-...
```

**Switch wiki generation model** ÔÇö edit `reposage/generator/wiki.py`:

```python
# Line ~100 ÔÇö module doc generation (fast, cheap)
message = self.client.messages.create(
    model="claude-haiku-4-5-20251001",   # ÔćÉ change this
    max_tokens=1500,
    ...
)

# Line ~130 ÔÇö architecture doc generation
message = self.client.messages.create(
    model="claude-haiku-4-5-20251001",   # ÔćÉ change this
    max_tokens=2000,
    ...
)
```

**Switch `ask` tool model** ÔÇö edit `reposage/mcp/server.py`:

```python
# Line ~290
message = client.messages.create(
    model="claude-opus-4-6",   # ÔćÉ change this
    max_tokens=1024,
    ...
)
```

**Available Claude models:**

| Model ID | Speed | Cost | Best for |
|----------|-------|------|---------|
| `claude-haiku-4-5-20251001` | Fastest | Lowest | Wiki generation (many API calls) |
| `claude-sonnet-4-6` | Fast | Medium | Balanced quality |
| `claude-opus-4-6` | Slower | Higher | `ask` tool, complex Q&A |

### Embedding model

RepoSage uses **ChromaDB's default embedding** (`all-MiniLM-L6-v2`, runs locally, no API key).

To switch to OpenAI embeddings, edit `reposage/storage/vector_store.py`:

```python
# Replace the default embedding function
import chromadb.utils.embedding_functions as ef

self._collection = self._client.get_or_create_collection(
    name=self.collection_name,
    embedding_function=ef.OpenAIEmbeddingFunction(
        api_key=os.environ["OPENAI_API_KEY"],
        model_name="text-embedding-3-small",
    ),
    metadata={"hnsw:space": "cosine"},
)
```

### Use a different LLM provider (OpenAI, Gemini, etc.)

Replace the Anthropic client in `reposage/generator/wiki.py` and `reposage/mcp/server.py`:

**OpenAI:**
```python
from openai import OpenAI
client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": prompt}],
)
content = response.choices[0].message.content
```

**Google Gemini:**
```python
import google.generativeai as genai
genai.configure(api_key=os.environ["GOOGLE_API_KEY"])
model = genai.GenerativeModel("gemini-2.5-flash")
response = model.generate_content(prompt)
content = response.text
```

## ­čöŹ How It Works

```
Source files (.m / .h / .swift / .java)
  Ôöé
  Ôľ╝ tree-sitter AST parsing
  Ôöé  ÔöťÔöÇ ObjC: @interface, @implementation, method selectors, message sends
  Ôöé  ÔöťÔöÇ Swift: class/struct/protocol/func, call_expression chains
  Ôöé  ÔööÔöÇ Java: class/interface/method, method_invocation
  Ôöé
  Ôľ╝ Symbol extraction Ôćĺ SQLite (symbols + relations tables)
  Ôöé  ÔöťÔöÇ Symbols: id, name, type, file, line, signature, doc_comment
  Ôöé  ÔööÔöÇ Relations: CALLS / EXTENDS / IMPLEMENTS / CONFORMS_TO / HAS_METHOD
  Ôöé
  Ôľ╝ Relation resolution (target_name Ôćĺ target_id cross-file)
  Ôöé
  Ôľ╝ Module clustering (group by top-level directory)
  Ôöé
  Ôľ╝ Embedding (ChromaDB, all-MiniLM-L6-v2, local, no API key)
  Ôöé
  Ôľ╝ Agent index generation (.reposage/*.json / .yaml)
  Ôöé
  Ôľ╝ Wiki generation (Claude API Ôćĺ docs/ARCHITECTURE.md + docs/modules/*.md)
  Ôöé
  Ôľ╝ MCP server (stdio, JSON-RPC)
       ÔööÔöÇ Claude Code / Cursor / Codex queries the knowledge graph
```

## ÔŁô Troubleshooting

**`No module named 'tree_sitter'`**
```bash
pip install tree-sitter==0.21.3 tree-sitter-languages tree-sitter-swift
```

**`Repository not indexed yet`**
```bash
reposage analyze /path/to/repo --skip-wiki
```

**`ANTHROPIC_API_KEY not set` ÔÇö wiki skipped**

Set the environment variable before running:
```bash
export ANTHROPIC_API_KEY=sk-ant-...
reposage wiki /path/to/repo
```

**Symbols show as `unknown`**

This affects ObjC methods with no-argument selectors in some edge cases. Run `--force` to re-index:
```bash
reposage analyze /path/to/repo --force --skip-wiki
```

**ChromaDB embedding model download is slow**

The `all-MiniLM-L6-v2` model (~80MB) is downloaded on first use to `~/.cache/chroma/`. Subsequent runs use the cached model.
